Welcome to Compiler Visualizer

A compiler is a program that transforms source code written in a high-level language like C or Java into low-level machine code that a computer can execute. This transformation involves several structured stages, each responsible for a specific task in the translation process.

This tool helps you understand the inner workings of a compiler by visualizing each of its core phases. Here’s a breakdown of what happens at every stage:

  • 1. Lexical Analysis: Scans the code and converts it into tokens such as identifiers, keywords, numbers, and operators.
  • 2. Syntax Analysis: Analyzes the token sequence to ensure it follows the language’s grammatical structure. Generates a parse tree.
  • 3. Semantic Analysis: Validates the logic of the program—checking data types, undeclared variables, scope rules, etc.—and builds the symbol table.
  • 4. Intermediate Code Generation: Produces a simplified, platform-independent version of the code suitable for further translation.
  • 5. Optimization: (Optional) Improves the efficiency of the intermediate code by eliminating redundancies and improving performance.
  • 6. Code Generation: Converts the optimized code into target machine or assembly code that can be executed by the processor.

With this visualizer, you'll be able to see how a simple C program flows through these phases. Let's explore how source code becomes machine-executable!
